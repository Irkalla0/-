# -
 用 STM32 + FreeRTOS 搭建面向机器人设备端的多传感器采集终端，完成采集、滤波、异常检测、通信上传、参数配置与上位机可视化闭环。
 同时覆盖 MCU/RTOS/驱动/协议/Python 工具链/软硬件联调/指标测试，是非常适合嵌入式秋招展示系统能力的项目。
 项目场景： 面向机器人设备、智能终端与工业采集节点，对设备运行、环境与电源状态进行连续监测，并在异常发生时本地告警与远程上报。
 最终效果： 设备端稳定完成多路异构传感器采样；系统内完成滤波、缓存、异常判断；通过 UART/CAN 与上位机通信；上位机可查看波形、设置参数、导出日志。

硬件方案
•	主控建议使用 STM32F407ZGT6 或 STM32F429，资源足够、资料丰富，适合 FreeRTOS、多外设与 DMA 场景。
•	传感器推荐组合：IMU（MPU6050/ICM42688）+ 温湿度（SHT30/AHT20）+ 电压电流（INA219 或 ADC 分压采样）+ ToF（VL53L0X）。
•	通信链路采用 UART 作为基础调试与上位机链路，CAN 作为增强模块，体现工业/机器人设备方向适配能力。
•	供电与接口设计考虑去耦、电源监测、下载调试口、接口扩展、基础防反接与 ESD 保护。
软件架构设计
层级	模块	职责
BSP 层	GPIO / UART / I2C / SPI / ADC / DMA / TIM / CAN	完成底层初始化与统一接口封装
Driver 层	imu_drv / sht30_drv / ina219_drv / tof_drv	每个传感器独立 init、read_raw、convert、self_test
Service 层	滤波、环形缓冲、CRC16、协议封包、参数管理、日志系统	沉淀可复用公共能力
Task 层	sensor_task / process_task / comm_task / log_task / watchdog_task	完成多任务采样、处理、通信与健康管理
App 层	状态机、阈值、告警策略、模式切换	形成完整业务逻辑
推荐任务划分：sensor_task 负责异步采样，process_task 负责滤波与异常检测，comm_task 负责协议收发，log_task 负责日志缓存，watchdog_task 负责系统心跳和故障恢复。
